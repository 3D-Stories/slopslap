## Findings

### 1. [High] correctness · high confidence — scripts/eval/candidates.py `_SLOPSLAP_REPAIRS`; tests/test_eval_run.py `test_kukakuka_real_repair_zero_invariant_violations`

> +    "kukakuka-prd": [

> +         b"Two surfaces"),                                      # drop the inflated "materials" metaphor

The asserted “model-driven” repair is actually supplied by exact, fixture-specific byte replacements in `_SLOPSLAP_REPAIRS`. This evaluation can prove that the edit application and invariant checks accept two preselected edits, but it cannot prove that a model or generalized repair mechanism identified or repaired the harm.

**Recommendation:** Change the test and results language to describe this as a seeded fixture-specific repair, or replace the `_SLOPSLAP_REPAIRS["kukakuka-prd"]` entries with output produced through the actual model-driven candidate-generation path and test that path.

### 2. [High] internal-consistency · high confidence — docs/reviews/2026-07-12-slopslap-eval-results.md, kukakuka-prd results

> +- slopslap disposition: **repair** — repaired 2 demonstrated-harm passage(s)
>  - slopslap correctly ABSTAINED on clean distinctive prose (keystone rule: edit only demonstrated harm) → zero invariant violations, zero flattening. The repair CAPABILITY is proven on the 3 seeded canonical fixtures above.

The results document now reports that kukakuka was repaired, but its conclusion still says the same document was clean and that SlopSlap abstained. Readers therefore receive mutually exclusive evaluation outcomes, and the claimed keystone-rule result is false under the changed run.

**Recommendation:** Update the kukakuka-prd conclusion in `docs/reviews/2026-07-12-slopslap-eval-results.md` to state that two localized repairs were made, distinguish those repairs from the flag-only cadence diagnoses, and remove the clean-prose abstention claim.

### 3. [Medium] completeness · high confidence — tests/test_eval_run.py `test_kukakuka_real_repair_zero_invariant_violations`

> +- slopslap disposition: **repair** — repaired 2 demonstrated-harm passage(s)
> +    assert k["disposition"] == "repair" and k["edits"] >= 1

The documented result requires two repairs, but the regression test accepts only one. If either exact replacement stops resolving, the test still passes while the stated two-passage result and intended coverage have regressed.

**Recommendation:** Change `test_kukakuka_real_repair_zero_invariant_violations` to assert `k["edits"] == 2` and verify both expected resulting passages, or revise the documented success criterion so it does not promise exactly two repairs.

### 4. [Medium] consistency · high confidence — scripts/eval/run_eval.py `_kukakuka` result construction

> +        "changed_bytes": sum(len(e.replacement) + (e.end_byte - e.start_byte) for e in cand.edits),
> +        "preservation": {"changed_byte_ratio": round(sum(len(e.replacement) for e in cand.edits) / max(1, len(prd)), 6),

The two similarly named change-size metrics use incompatible numerators: `changed_bytes` adds removed-span and replacement lengths, while `changed_byte_ratio` counts only replacement lengths. Consequently the reported ratio is not the ratio corresponding to the reported changed-byte total, making the preservation evidence internally misleading.

**Recommendation:** Define one byte-change measure in `run_eval.py` and use the same numerator for both fields. If insertion-plus-deletion volume is intended, rename it to `edit_volume_bytes` and compute `changed_byte_ratio` from that value; otherwise compute both from a documented diff-byte definition.
