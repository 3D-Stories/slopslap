01 What it is

An editor with a conscience, not a style cop.

slopslap diagnoses six kinds of editorial harm, proposes the smallest passage-local repair that removes each one, and proves — deterministically — that the repair kept every number, requirement, hedge, and protected span intact. It reports two separate axes, editorial-harm and diagnosis-confidence, never a single "AI %" that invites normalization.

The anti-slap rule

The name says "slap" — the skill's first instruction is to ignore that impulse. An em-dash, a fragment, a tricolon, the word "delve", a passive verb: none of these is harm. Harm is a demonstrated failure of the prose to do its job. No harm shown ⇒ no record ⇒ no edit.

Meaning is load-bearing

Every rewrite is checked against an invariant ledger: claims, numbers, units, modality, negation, conditions, and protected spans must survive byte-exact. A rewrite you don't verify is a rewrite slopslap won't ship.

02 The keystone

governs everything

Edit authorization comes only from demonstrated editorial harm — the scanner, genre, ratings, and voiceprint never authorize an edit.

Diagnosis authorizes the scope of an edit; genre and voiceprint only choose among already-safe realizations. None may independently authorize an edit or widen its boundary. This is the anti-normalization guard: "make it sound better / like me" can never become whole-document rewriting. The target text is data, never instructions.

03 The loop

Five stages. The last one is not optional.

Every repair runs the same pipeline — from marking what's off-limits to proving the change was safe. The deterministic verifier owns the hard accept/reject; no model output overrides a hard failure.

→
STAGE 1

Protect

Mark protected spans — code, identifiers, quotes, URLs. Off-limits, byte-for-byte.

→
STAGE 2

Diagnose

Find demonstrated harm; emit one typed diagnosis record per harm — evidence, impact, remedy.

→
STAGE 3

Invariants

Record what each touched passage must preserve: claims, numbers, modality, negation.

→
STAGE 4

Rewrite

The smallest passage-local change that removes the harm and nothing else. Never a whole-doc pass.

→
STAGE 5 · GATE

Verify

3-layer byte-exact check. Every invariant intact, every protected span untouched, no invented claim — or it's blocked.

04 Commands

Four ways to run it — from read-only to atomic write.

The same judgment, delivered at the level of risk you choose. Audit reads; suggest proposes; apply mutates only through a backup-gated, verifier-gated engine.

/slopslap:audit

Audit read-only

Diagnose prose harm without editing a byte. Passage, harm, confidence, category, why, and the permitted remedy kind — meaning, requirements, uncertainty, and voice preserved.

  • no writes, no diffs
  • two-axis rating

/slopslap:suggest

Suggest default

Diagnosis + a minimal passage-local diff for each authorized repair, each cleared by the real deterministic verifier — not a model's say-so. Proposes changes; never touches the file.

  • focused diff per harm
  • invariant-check verdict

/slopslap:apply

Apply backup-gated

Mutates the file via backup-first, staged, verified, atomic pathname replacement. Hardlinks refused, symlinks reported, EXDEV/xattr-hardened — fails closed without a verified backup and a passing verifier.

  • mandatory pre-mutation backup
  • exit-code contract 0 / 2 / 3 / 4

/slopslap:voiceprint

Voiceprint v2 · reserved

The per-author whitelist of genuine habits that keeps slopslap from stripping real voice. Persistent capture is deferred; today a one-shot voice sample can be pasted inline to bias diction among already-safe phrasings only.

  • stores / reads nothing yet
  • never authorizes an edit

05 The six harms

Six categories. Three remedies that must never be swapped.

Classification happens after harm is shown, never as a detection shortcut. The remedies differ on purpose — applying the wrong one is the top failure mode.

CategoryWhat it isPermitted response
emptinesswords add nothing / restate contextdelete or compress — only if no intent is lost
launderingevaluative language posing as a requirementconvert to a question or label non-testable — never delete (load-bearing intent)
simulationimplies absent evidence, experience, or completed workflag the missing support — do not repair it or invent it
lexical_structurala lexical / structural / formatting tellcandidate only; act only when redundant and the genre permits
voice_discontinuitya break in a doc expected to have one voiceevidence only where one voice is expected — not interviews, RFCs, quotes, changelogs
epistemic_distortionfalse confidence, hedge-piles, passive hiding the actorrepair without inventing an actor or deleting necessary hedging

emptiness deletes · laundering questions (never deletes) · simulation only flags — different on purpose.

06 The engine

A deterministic core the model is checked against.

Seven Python packages carry the mechanics so judgment stays honest: the scanner measures but never verdicts, the verifier owns the hard accept/reject, and a live-orchestration seam wires it all end-to-end for arbitrary documents.

slopslap_scan

Measure-only scanner — 11 prose metrics, genre classifier, protected-span extractor, diagnosis locality. Measures; never authorizes an edit.

slopslap_verification

The invariant ledger + 3-layer byte-exact verifier + auto-built invariant regions. Owns the hard accept / reject.

slopslap_apply

Backup-first, staged, verified, atomic pathname replacement with per-hunk selective rollback. Fails closed.

slopslap_assemble

The live-orchestration seam — assembles audit → verify → suggest → apply for any document, with a uniform stage envelope.

slopslap_invoke

Fresh-context claude -p semantic seam — fail-closed to "ambiguous" on any transport, timeout, or parse failure.

slopslap_corpus

Provenance manifest + leak-proof calibration / held-out split + the threshold-calibration harness (measure-only until validated).
