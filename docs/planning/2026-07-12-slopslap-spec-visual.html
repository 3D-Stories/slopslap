slopslap.

Repairs prose that carries high editorial cost — genericness, unsupported claims, synthetic cadence, hidden responsibility — while preserving meaning, requirements, uncertainty, and your real voice. It is not an AI-authorship detector, and it does not treat style as contamination.

01 · the reframe

Editorial cost, not AI-detection

Detecting "was this written by AI" is unreliable and aims at the wrong target. slopslap instead finds where language pretends to carry meaning and repairs it — recovering intended content when the source supports it, and making missing content visible when it doesn't. That reframe is what lets it beat the humanizer skill, whose core flaw is treating stylistic features (em-dashes, lists, parallelism, passive voice) as contaminants regardless of genre or author.

02 · the rule that governs everything

Keystone

Edit authorization comes from demonstrated editorial harm. Invariants constrain meaning; genre constrains function; the voiceprint only chooses among safe phrasings. Diagnosis authorizes scope; genre and voiceprint authorize realization — neither may independently authorize an edit or widen its boundary.

This single rule is the anti-normalization guard: "make it sound like me" can never become whole-document rewriting.

03 · pipeline

The loop

protect→diagnose→establish invariants→rewrite (minimal, passage-local)→verify

The invariant ledger is append-only during rewrite — new invariants may be added, never weakened or silently removed. The rewrite pass never verifies its own output.

04 · what counts as a tell

Taxonomy — categories kept separate

Clustered confidence: no single tell is dispositive. The critical discipline is not collapsing these three — each needs a different remedy, or "empty" becomes a license to delete load-bearing intent.

ConditionPermitted response
Semantic emptiness — words add nothing / repeat contextdelete or compress — only if no intent is lost
Specification-laundering — evaluative language posing as a requirement ("intuitive", "seamless", "unforgettable")convert to a question or label non-testable — do not delete (it's real intent)
Content-simulation — implies absent evidence / examples / analysisflag the missing support — do not repair substantively
Lexical / structural / rhetorical / formatting tellscandidates only; act only when redundant AND genre permits
Voice-discontinuityevidence only when the doc is expected to have one voice (not interviews, RFCs, quotes, changelogs)
Epistemic distortion — false confidence, hedge-piles, actor-hiding passiverepair without inventing an actor or deleting necessary hedging

Two ratings, never a single "AI %": editorial-harm low/med/high · diagnosis-confidence low/med/high. One number invites threshold-driven over-editing.

05 · the safety spine

Invariant ledger + 3-layer verify

decidedMachine-readable ledger

JSON, shared by rewrite + verify. Closed kind enum (literal, number, normative_statement, condition, exception, causal_claim, attribution, defined_term, cross_reference, unsupported_intent, missing_support, intentional_repetition, protected_span) + closed preservation enum (byte / lexically / semantic / relationship exact, or surface_only = "preserve that it's unresolved", not the vague words).

decidedCanonical coordinates · r3 fix · WF5 #3

All positions — including the ledger's source — use original-byte offsets as the one canonical system (line numbers are derived display-only) + an original→revision edit map. Without it, multi-hunk verification and the per-hunk rollback misattribute passages that shifted when an earlier hunk changed length.

decided3-layer verify · WF5 #7

(1) deterministic integrity — hashes, numbers, modality inventory, balanced fences, edits-in-range; this layer is code and owns the hard accept/reject — no model output overrides it. (2) extraction-then-compare — re-extract into the same ledger shape, match. (3) adversarial semantic check in a fresh context that never saw the rewriter's reasoning (may be a different model). The rewriter never grades itself.

decidedFailure decision rule

Hard (protected-span mutation, changed number/modality, invented claim, dropped condition) → auto-reject. Ambiguous → surface, don't apply. Soft style → keep safer version, never escalate to satisfy style. Ledger uncertainty → ask. Per-hunk rollback, never whole-doc.

06 · signal, not verdict

Scanner — scan_prose.py

Measures, never decides. Uses a CommonMark parser (excludes code, HTML, blockquotes, URLs, inline code; keeps headings + visible link text with structural type). Emits per metric: eligible-units, count, rate, locations, soft-flag. ~8 reliable metrics in MVP: sentence-length distribution, sentence_length_dispersion (not "burstiness" — that implies a validated authorship feature), repeated openers, transition clusters, punctuation rates, paragraph sentence-count runs, vague-attribution clusters, bold-label density, stock lexical clusters. Cadence-similarity flags only with a corroborating signal — its exact features/weights (and every metric's parameters) are pinned in scanner-metrics.md with fixtures before it ships (WF5 #6). No cross-document percentiles in MVP (needs a genre reference corpus). Thresholds are versioned triage aids — purpose: candidate_selection_only.

gpt-soul consult · WF5 #2/#5/#8Markdown dependency: capability-gate + vendor

Plugins have no managed venv, so a bare import markdown_it would fail for most users. Resolution: gate by explicit --format text|markdown (never sniffed); plain-text is stdlib-only + always on; Markdown is either parsed by a conforming parser or reported unavailable — never routed through the plain-text path.

decidedFail loud, valid JSON

On an absent/incompatible parser the scanner emits valid JSON {"status":"capability_unavailable",…,"metrics":null} + stderr + a dedicated advisory-skip exit code 10 (0=ok, 1=defect) the caller turns into "skip these metrics". Never partial metrics that read as authoritative.

decidedVendor, never runtime-pip

Real Markdown support = vendored pinned markdown-it-py + mdurl under a plugin-private vendor/python/, with licenses, provenance, and CommonMark fixtures on Linux/macOS/Windows. Rejected: runtime pip-install and a home-grown stdlib subset (the exclusion rules cross block+inline — a "subset" is just an approximate parser under another name).

WF5 #8Bare-URL = post-tokenization rule

Bare-URL exclusion is a deterministic post-tokenization matcher bound to explicit extraction fixtures — not a dependency on a parser linkify extension — so counts/locations stay reproducible across parser versions. Since slopslap's inputs (specs, PRDs, essays) are Markdown, the vendored parser is in MVP.

07 · context awareness

Genre — per-region, asymmetric failure

Not a whole-document label. Hierarchical: document_genre + regions[], segmented at headings/blocks. Precedence: explicit user declaration > file/repo context > structural markers > content inference. Profiles: general-prose · technical-doc · spec (repetition = correctness infrastructure — preserve) · legal (flag, audit-only) · PRD (challenge adjectives-posing-as-requirements) · marketing · personal (voice weighted high). Asymmetric failure: low confidence → the most preservation-heavy applicable profile (stiffening marketing < changing a spec). Genre shapes diagnosis + rewrite constraints; it never decides prose is "bad."

08 · how it's used

Three modes

MVPaudit

Read-only diagnosis: passage → harm → confidence → class → why → repair kind → missing evidence. No edits.

MVP · defaultsuggest

Diagnosis + focused diff + questions only for facts that block a specific repair + invariant-check. Placeholders proposed outside the doc (inserting [DEFINE X] can make an invalid doc look complete).

MVPapply

In-place, explicit request, gated by a mandatory pre-mutation backup written to a user-local dir outside the repo by default — an in-tree override must create + verify an ignore rule and fail closed if it can't (WF5 #4: an in-tree dir is not git-ignored just by living there). Universal safety net, git or not. Per-hunk selective rollback is now in MVP (reuses the edit-map machinery verification already needs).

09 · build order

MVP cut

Sol vetoed audit-only: it proves the skill can name defects, not safely repair them — the hard part. So the MVP exercises the whole protect→verify loop, minus weakly-validated personalization. Build order is fixtures-first: pin the 3 canonical eval fixtures (the red test) before scaffolding, then the plugin, then the scanner, then the eval loop.

Ships in v1

  • audit + suggest modes
  • passage-local edits only
  • protected-span extraction
  • minimal invariant ledger
  • deterministic + separate semantic verify
  • ~8 reliable scanner metrics (markdown via vendored CommonMark parser)
  • 3 genre profiles (general, PRD/spec, personal)
  • 3 canonical evals + clean-document controls
  • apply mode — backup-gated
  • byte-offset edit-map + per-hunk selective rollback
  • manual voice sample (no learning)

Deferred, in order

  • 1 · mixed-region genre classification
  • 2 · expanded scanner heuristics
  • 3 · persistent voiceprint learning
  • 4 · legal profile (after dedicated tests)

10 · the learned voice — v2

Voiceprint is defensive

The strong framing (Claude's independent pass, confirmed by Sol): in a slop-stripping tool the voiceprint's primary job is a per-author whitelist of genuine habits — your em-dashes, fragments, "honestly" — so slopslap doesn't strip real voice as if it were AI-tell. It's permissive ("keep this, it's his"), not generative ("write like him"). That's exactly what humanizer can't do.

Source honesty

Prompts teach diction / directness / hedging but are blind to long-form rhythm and full of mode-artifacts (typos, lowercase) that must never become prose style. Evidence weights: user long-form 1.0 · edits-to-a-rewrite 0.9 · accepted-as-voice 0.6 · prompt shell 0.35 · unchallenged output 0.0 · pasted 0.0.

Authority order

protected spans > invariant ledger + no-fabrication > genre > current explicit instruction > voiceprint > neutral default. "Make this formal" beats a casual history; nothing overrides meaning.

Anti-caricature

Trait must recur across ≥3 sessions; never inject a tic just because frequent; on voiceprint-vs-local-document conflict prefer local continuity; idempotence — repeat runs must not amplify contractions / dashes / directness.

Consent

Opt-in, local-only, inspectable + editable (show / disable / reset / export / delete). Stores derived counts + approved examples, never raw prompts. Needs a companion opt-in UserPromptSubmit hook to capture.

11 · what could break

Second-order failure modes

Beyond the obvious. These go in eval-cases.md as adversarial fixtures — the failures that make a "clean" edit actively harmful.

signal→authorization

The riskiest one: a scanner soft-flag / genre / voiceprint silently becomes permission to edit. The keystone rule + the guard test exist for exactly this.

voice erosion

Iterative sanding (runs progressively flatten voice), non-native / dialect normalization, dialect suppression. Guarded by idempotence + "leave passed passages alone."

meaning drift

Competence-laundering (cleaner prose making weak reasoning look authoritative), uncertainty deletion, responsibility reassignment (invented actor), diff-fragmentation (safe micro-edits jointly shift tone).

self-slop + over-reach

Diagnosis theater, taxonomy leakage (outputting "specification-laundering" vs plain language), vision-policing (challenging every PRD aspiration), question-explosion, the audit itself reading as slop.

12 · proof

Evals

Programmatic: protected spans byte-identical; URLs / identifiers / defined-terms exact; numeric multiset preserved; modal inventory + negation + condition markers preserved; no edits outside authorized ranges; no invented numbers / names / citations / placeholders; Markdown parses pre+post; idempotence (run-2 = no material diff); scanner JSON schema-valid. LLM-judge (blinded A/B, cites spans): meaning + actor preservation, unsupported-claim introduction, unresolved-intent stays visible, edit locality, seeded defect fixed without normalizing surrounding distinctive prose. Baselines: humanizer AND original-unchanged (unchanged should win on clean/stylized prose). Clean-document controls — a system that always finds something to fix fails these.

Decision rule — reproducible pass/fail (WF5 #1)

Hard gates: any programmatic failure = fixture fail, overriding every judge score. Each LLM-judge dimension scored 0/1/2 over ≥3 blinded trials, taking the per-dimension median. A fixture beats a baseline iff it clears all hard gates, its median dimension-sum ≥ the baseline's, and it wins ≥1 dimension with none worse by >1; ties resolve to the more preservation-heavy output. On clean-document controls the pass criterion is abstention — original-unchanged must win or tie. Engine choice = the tier clearing the most fixtures' hard gates; per-tier results recorded, never one blended number.

The 3 canonical fixtures (to pin in eval-cases.md — FIRST)

distinctive human essay w/ 2 seeded generic paragraphs · normative spec w/ 1 vague non-requirement · underspecified PRD w/ real numeric constraints + missing decisions.
